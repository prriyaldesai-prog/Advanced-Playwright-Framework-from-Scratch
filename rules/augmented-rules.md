# Augmented Rules - Advanced Testing Patterns & Enhancements

## Overview
These augmented rules extend the core quality framework with advanced patterns, AI-assisted best practices, and enhanced capabilities for the Playwright testing framework.

## 1. AI-Assisted Code Generation Rules

### When Using AI Assistants (Copilot, Claude)
- ✅ Always review generated code for correctness
- ✅ Ensure generated tests include proper tags (@p0, @p1, @e2e)
- ✅ Verify TypeScript types are explicit (no `any`)
- ✅ Check that logger is used instead of console.log
- ✅ Validate all assertions are meaningful
- ✅ Ensure tests are independent and can run in parallel
- ✅ Review for hardcoded values (should use .env)
- ✅ Add comments if logic is complex

## 2. Advanced Testing Patterns

### Page Object Model (POM)
Recommended for complex applications with multiple pages.

**Structure:**
```
src/pages/
├── BasePage.ts
├── LoginPage.ts
├── DashboardPage.ts
└── ProfilePage.ts
```

**Example:**
```typescript
// src/pages/BasePage.ts
import { Page } from '@playwright/test';
import logger from '@utils/logger';

export class BasePage {
  constructor(protected page: Page) {}

  async goto(url: string): Promise<void> {
    logger.info(`Navigating to ${url}`);
    await this.page.goto(url);
  }

  async getTitle(): Promise<string> {
    return await this.page.title();
  }
}

// src/pages/LoginPage.ts
import { BasePage } from './BasePage';

export class LoginPage extends BasePage {
  async login(email: string, password: string): Promise<void> {
    logger.info('Logging in user');
    await this.page.fill('input[name="email"]', email);
    await this.page.fill('input[name="password"]', password);
    await this.page.click('button[type="submit"]');
    await this.page.waitForURL('/dashboard');
  }
}
```

### Fixtures for Reusable Test Setup
```typescript
import { test as base } from '@playwright/test';

interface TestFixtures {
  authenticatedPage: Page;
  demoUser: { email: string; password: string };
}

export const test = base.extend<TestFixtures>({
  demoUser: {
    email: 'test@example.com',
    password: 'password123',
  },

  authenticatedPage: async ({ page, demoUser }, use) => {
    logger.info('Setting up authenticated page');
    await page.goto('/login');
    await page.fill('input[name="email"]', demoUser.email);
    await page.fill('input[name="password"]', demoUser.password);
    await page.click('button[type="submit"]');
    await page.waitForURL('/dashboard');

    await use(page);

    logger.info('Cleaning up authenticated session');
    await page.close();
  },
});
```

**Usage:**
```typescript
import { test } from './fixtures';

test('@p0 @e2e dashboard displays user info', async ({ authenticatedPage, demoUser }) => {
  logger.info(`Verifying dashboard for ${demoUser.email}`);
  await expect(authenticatedPage.locator('[data-testid="user-name"]')).toContainText(demoUser.email);
});
```

## 3. Performance Optimization Rules

### Performance Testing
```typescript
import { test, expect } from '@playwright/test';
import logger from '@utils/logger';

test('@e2e page load performance', async ({ page }) => {
  logger.info('Measuring page load performance');

  const startTime = Date.now();
  await page.goto('/dashboard', { waitUntil: 'networkidle' });
  const loadTime = Date.now() - startTime;

  logger.info(`Page loaded in ${loadTime}ms`);
  expect(loadTime).toBeLessThan(3000); // Must load in < 3 seconds
});
```

### Performance Benchmarks
| Metric | Target | Action |
|--------|--------|--------|
| Page load | < 3 seconds | Optimize if slower |
| API response | < 500ms | Check backend |
| Test execution | < 30s per test | Profile test |
| All tests | < 5 minutes | Review parallelization |

### Performance Profiling
```typescript
test('@e2e API call performance', async ({ request, page }) => {
  logger.info('Profiling API performance');

  const times: number[] = [];

  for (let i = 0; i < 5; i++) {
    const start = Date.now();
    await request.get('/api/data');
    times.push(Date.now() - start);
  }

  const avgTime = times.reduce((a, b) => a + b) / times.length;
  logger.info(`Average API response: ${avgTime}ms`);
  expect(avgTime).toBeLessThan(500);
});
```

## 7. Error Recovery & Retry Patterns

### Retry on Transient Failures
```typescript
test('@p0 @e2e handles transient network errors', async ({ page }) => {
  logger.info('Testing error recovery');

  const maxRetries = 3;
  let attempt = 0;

  while (attempt < maxRetries) {
    try {
      logger.info(`Attempt ${attempt + 1}/${maxRetries}`);
      await page.goto('/api-endpoint', { timeout: 5000 });
      break;
    } catch (error) {
      attempt++;
      if (attempt === maxRetries) {
        logger.error('Max retries exceeded');
        throw error;
      }
      logger.warn(`Retry after ${1000 * attempt}ms`);
      await page.waitForTimeout(1000 * attempt);
    }
  }
});
```

### Graceful Degradation
```typescript
test('@p1 @e2e handles missing optional elements', async ({ page }) => {
  logger.info('Testing graceful degradation');
  await page.goto('/dashboard');

  const optionalElement = page.locator('[data-testid="premium-badge"]');
  const isVisible = await optionalElement.isVisible().catch(() => false);

  if (isVisible) {
    logger.info('Premium badge is visible');
    await expect(optionalElement).toBeVisible();
  } else {
    logger.info('Premium badge is hidden (expected for non-premium users)');
  }
});
```

## 13. Advanced Assertions

### Custom Matchers
```typescript

  async toBeAccessible(page: Page) {
    // Custom accessibility check

      return document.querySelectorAll('[role]').length > 0;

      pass,
      message: () => 'Page should have ARIA roles',
```
