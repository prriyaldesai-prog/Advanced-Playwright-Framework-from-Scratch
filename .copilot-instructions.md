# GitHub Copilot & Claude Instructions

## Playwright Framework Quality Rules

You are an expert assistant helping with the **Advanced Playwright Framework from Scratch** project. When assisting with this project, strictly adhere to the following rules and standards:

---

## 1. Code Quality Standards

### Linting Rules
- **No `any` types**: Always use proper TypeScript types
- **No console.log**: Use the Winston logger from `@utils/logger` instead
- **No unused code**: Remove unused imports, variables, and functions
- **Max line length**: 120 characters
- **Quote style**: Single quotes (')
- **Indentation**: 2 spaces
- **Variables**: Use `let`/`const`, never use `var`
- **No trailing whitespace**

**Logger Usage:**
```typescript
import logger from '@utils/logger';
logger.info('Message');
logger.warn('Warning');
logger.error('Error');
logger.debug('Debug info');
```

### Code Formatting
- **Print width**: 100 characters
- **Tab width**: 2 spaces
- **Single quotes**: Always
- **Trailing commas**: es5
- **Semicolons**: Always add them
- **Arrow parentheses**: Always add them: `(x) => x`
- **Bracket spacing**: `{ foo: bar }`

### TypeScript Standards
- **Strict mode**: Always enabled
- **No implicit any**: Explicitly type all parameters
- **Return types**: All functions must have explicit return types
- **Interfaces**: Use interfaces for object shapes
- **Enums**: Use enums for constants
- **No type assertions**: Unless absolutely necessary with justification

**Example:**
```typescript
interface UserData {
  name: string;
  email: string;
  age: number;
}

function validateUser(user: UserData): boolean {
  return user.email.includes('@');
}
```

## 2. Test Structure & Standards

### Test File Location & Naming
- **Location**: `src/tests/`
- **Extension**: `.spec.ts`
- **Naming**: kebab-case (e.g., `login.spec.ts`)
- **Organization**: Group by feature in subfolders

### Test Tags (MANDATORY)
Every test MUST have one of these tags:
- `@p0` - Critical tests (must always pass)
- `@p1` - Important tests (should pass before merge)
- `@e2e` - End-to-end tests
- Optional: `@smoke` for quick validation

**Example:**
```typescript
test('@p0 @e2e user login with valid credentials', async ({ page }) => {
  // Test implementation
});
```

### Test Requirements
- ✅ Descriptive test names (what, not how)
- ✅ Independent tests (no shared state)
- ✅ Proper setup and teardown
- ✅ Meaningful assertions
- ✅ Logging of key steps
- ✅ No hardcoded wait times (use `waitFor` instead)
- ✅ Parallel execution compatible

### Test Template
```typescript
import { test, expect } from '@playwright/test';
import logger from '@utils/logger';

test.describe('@e2e Login Feature', () => {
  test.beforeEach(async ({ page }) => {
    logger.info('Setting up test');
    await page.goto('/login');
  });

  test('@p0 should login successfully', async ({ page }) => {
    logger.info('Testing login with valid credentials');
    await page.fill('input[name="email"]', 'test@example.com');
    await page.fill('input[name="password"]', 'password123');
    await page.click('button[type="submit"]');
    await expect(page).toHaveURL('/dashboard');
    logger.info('Login test passed');
  });

  test.afterEach(async ({ page }) => {
    logger.info('Cleaning up');
  });
});
```

## 3. Best Practices

### DO ✅
- ✅ Use page object models for complex tests
- ✅ Extract common test logic into fixtures
- ✅ Add logger calls at test start and key steps
- ✅ Use descriptive variable names
- ✅ Keep tests focused and short
- ✅ Use beforeEach/afterEach for setup/cleanup
- ✅ Await all async operations
- ✅ Use proper assertions
- ✅ Follow DRY principle
- ✅ Add comments for complex logic

### DON'T ❌
- ❌ Use hardcoded values (use .env instead)
- ❌ Add console.log calls (use logger)
- ❌ Share state between tests
- ❌ Use hardcoded sleep/setTimeout (use waitFor)
- ❌ Write tests that are order-dependent
- ❌ Use `any` types
- ❌ Ignore TypeScript errors
- ❌ Skip tests in commits
- ❌ Add sensitive data in code
- ❌ Commit with linting errors

## 8. Common Patterns

### Page Object Model
```typescript
// pages/LoginPage.ts
export class LoginPage {
  constructor(private page: Page) {}

  async navigate(): Promise<void> {
    await this.page.goto('/login');
  }

  async login(email: string, password: string): Promise<void> {
    await this.page.fill('input[name="email"]', email);
    await this.page.fill('input[name="password"]', password);
    await this.page.click('button[type="submit"]');
  }
}
```

### Fixtures for Reusable Setup
```typescript
test.beforeEach(async ({ page }) => {
  await page.goto('/');
  await page.fill('input[name="search"]', 'query');
});
```

### Proper Assertions
```typescript
// Good - specific assertion
await expect(page.locator('h1')).toHaveText('Welcome');

// Bad - vague assertion
expect(await page.textContent('h1')).toBeTruthy();
```

## 9. Logging Standards

### When to Log
- Test start and major milestones
- Before/after important actions
- On errors or unexpected behavior
- Setup and cleanup operations

### Log Levels
```typescript
logger.error('Critical error occurred');      // Errors
logger.warn('Something might be wrong');      // Warnings
logger.info('Test step executed');            // Information
logger.debug('Detailed debug information');   // Debug details
```

### Log Output
- Console: Color-coded, timestamp with seconds
- File (`logs/error.log`): JSON format, errors only
- File (`logs/combined.log`): JSON format, all messages

## 10. TypeScript Configuration

### Type Annotations
```typescript
// Good
function calculateTotal(items: Item[]): number {
  return items.reduce((sum, item) => sum + item.price, 0);
}

// Bad - avoid any
function calculateTotal(items: any): any {
  return items.reduce((sum, item) => sum + item.price, 0);
}
```

### Interfaces
```typescript
interface TestUser {
  email: string;
  password: string;
  name: string;
}

interface LoginResponse {
  success: boolean;
  token?: string;
  error?: string;
}
```

## 11. Error Handling

### Always Handle Errors
```typescript
try {
  await page.goto(url);
} catch (error) {
  logger.error(`Navigation failed: ${error}`);
  throw error; // Re-throw for test to fail
}
```

### Use Proper Assertions
```typescript
await expect(page.locator('.error')).toHaveText('Email is required');
```

## 15. Augmented Rules - Advanced Patterns

For advanced testing scenarios, refer to `rules/augmented-rules.md`:

### When to Use Augmented Patterns
- **Page Object Model**: Use for complex apps with multiple pages
- **Fixtures**: Use for reusable test setup and teardown
- **Data-Driven Testing**: Use for testing multiple scenarios
- **API Testing**: Use for API endpoint verification
- **Performance Testing**: Use for load time and response time checks
- **Accessibility Testing**: Use for WCAG compliance
- **Visual Regression**: Use for UI layout changes
- **Error Recovery**: Use for retry and resilience testing
- **Test Data Management**: Use for faker and builders
- **Mocking & Stubbing**: Use for API and service worker mocks

### Example: Page Object Model
```typescript
// src/pages/LoginPage.ts
export class LoginPage extends BasePage {
  async login(email: string, password: string): Promise<void> {
    logger.info('Executing login');
    await this.page.fill('input[name="email"]', email);
    await this.page.fill('input[name="password"]', password);
    await this.page.click('button[type="submit"]');
  }
}

// tests/auth/login.spec.ts
import { test } from '@playwright/test';
import { LoginPage } from '@pages/LoginPage';

test('@p0 @e2e successful login', async ({ page }) => {
  const loginPage = new LoginPage(page);
  await loginPage.navigate();
  await loginPage.login('test@example.com', 'password');
});
```

### Example: Data-Driven Testing
```typescript
const testCases = [
  { email: 'user1@test.com', password: 'pass1' },
  { email: 'user2@test.com', password: 'pass2' },
];

testCases.forEach(({ email, password }) => {
  test(`@p1 @e2e login as ${email}`, async ({ page }) => {
    logger.info(`Testing login for ${email}`);
    // Test implementation
  });
});
```

### Example: Accessibility Testing
```typescript
import { injectAxe, checkA11y } from 'axe-playwright';

test('@p1 @e2e page is WCAG compliant', async ({ page }) => {
  await page.goto('/dashboard');
  await injectAxe(page);
  await checkA11y(page);
});
```
